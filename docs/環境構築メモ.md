### Draw.io Integration の使い方

VSCode 内で draw.io 図を作成・編集できる拡張機能。

**ファイル形式:**

| 拡張子 | 特徴 |
|--------|------|
| `.drawio` | draw.io 専用形式（XMLベース） |
| `.drawio.png` | PNG画像として保存しつつ、draw.io の編集情報も内包 |
| `.drawio.svg` | SVG形式。Markdown から `![](xxx.drawio.svg)` で直接表示可能 |

**基本操作:**

1. エクスプローラーで右クリック → `新しいファイル` → `ファイル名.drawio.png` などで作成
2. ファイルをクリックすると VSCode 内で draw.io エディタが起動
3. 図を編集後、保存（`Ctrl + S`）するとファイルに反映される

**Markdown への埋め込み:**

`.drawio.png` または `.drawio.svg` は通常の画像として Markdown に貼り込める。

```markdown
![図の説明](images/図.drawio.png)
```

**便利な操作:**

プレビュー画面に表示された draw.io 画像を **右クリック →「画像を開く」** で、該当ファイルの draw.io エディタに直接ジャンプして編集できる。

**リンク付きノードを使う場合:**

draw.io のノードにリンクを設定しても、`.drawio.png` ではリンクが機能しない。**リンクを有効にするには `.drawio.svg` 形式で保存する必要がある。**

**SVGリンクの動作確認について:**

`.drawio.svg` に埋め込んだリンクは、**VSCode のプレビュー上および GitHub のプレビュー上では動作しない**。
リンクの動作確認には **docsify** を使ってローカルでブラウザ経由で表示する形で対応。

さらに、通常の Markdown 画像埋め込み（`![](xxx.drawio.svg)`）のままでは docsify 上でもリンクが機能しなかったため、`index.html` に docsify プラグインを実装することで対応した。

**`index.html` での対応内容（プラグイン）:**

ページ描画のたびに `.svg` 拡張子を持つ `<img>` 要素を検出し、以下の処理を行う。

1. SVG ファイルをテキストとして fetch で読み込み、DOM に変換する
2. SVG 内の `<a>` リンクを走査し、`.md` への相対パスを docsify が認識できるハッシュ形式（`#/xxx.md`）に書き換える
3. 元の `<img>` 要素を実際の `<svg>` 要素に置き換えてインライン展開する

### CSV 管理（Edit CSV + grid.js）

**Edit CSV 拡張機能**を使って CSV ファイルをスプレッドシート感覚で編集し、**grid.js** でブラウザ上に表テーブルとして表示する構成。

**運用フロー:**

1. CSV ファイルを `data/` フォルダに配置
2. VSCode で CSV ファイルを開くと Edit CSV が自動起動 → セル操作で編集可能
3. Markdown からコードブロック（言語指定 `gridjs`）＋ docsify のファイルインクルード構文で参照

```markdown
[filename](../data/ファイル名.csv ':include :type=code gridjs')
```

4. docsify（`python -m http.server 5000`）でブラウザ表示すると、検索・ソート・ページネーション付きのテーブルとして描画される

**grid.js の読み込みについて:**

`index.html` に `gridjs.umd.js` と CSV パース処理を含むカスタムプラグインを実装している。`docsify-gridjs-plugin` という npm パッケージは存在しないため、index.html 内のプラグインで代替している。

| OS | ショートカット |
|----|--------------|
| Windows | `Ctrl + Tab` |
| Mac | `Cmd + Option + ←` |

## 3. 設定 (settings.json)

```json
{
  "markdown.copyFiles.destination": {
    "**/*.md": "images/"
  }
}
```

**変更した設定とその理由:**

- `markdown.copyFiles.destination` — Markdown ファイルからクリップボード画像を貼り付けた際の保存先を指定する。
  - キー `"**/*.md"` : 対象ファイルのパターン（すべての .md ファイル）
  - 値 `"images/"` : すべての画像を `images/` フォルダにまとめて保存
  - 例: `画像張り込みテスト.md` に貼り付けた場合 → `images/image.png`
