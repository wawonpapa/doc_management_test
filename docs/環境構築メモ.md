### docsify によるブラウザ表示

**docsify** は Markdown ファイルをビルドなしでブラウザ上に描画する静的サイトジェネレーター。`index.html` 1 ファイルをリポジトリルートに置くだけで動作し、Markdown ファイルをそのまま管理できる。

- 公式サイト: https://docsify.js.org/
- テーマ: docsify-themeable（`theme-simple`）— https://jhildenbiddle.github.io/docsify-themeable/

**セットアップ:**

特別なインストールは不要。`index.html` に CDN リンクを記載するだけで動作する。ローカル確認は Python の HTTP サーバで可能。

```bash
python -m http.server 5000
# → http://localhost:5000 をブラウザで開く
```

**主な設定項目（`index.html` 内の `window.$docsify`）:**

| 設定キー | 値 | 説明 |
|---------|-----|------|
| `name` | サイト名 | サイドバー上部に表示されるタイトル |
| `repo` | GitHub URL | 右上にリポジトリリンクを表示 |
| `loadSidebar` | `true` | `_sidebar.md` をサイドバーとして読み込む |
| `subMaxLevel` | `0` | サイドバーに表示する見出しの深さ（0 = ファイル単位のみ） |
| `auto2top` | `true` | ページ遷移時に自動でスクロールトップへ戻る |

**`_sidebar.md` の書き方:**

Markdown のリスト記法でナビゲーションを定義する。フォルダ階層はネストで表現する。

```markdown
* [README](README.md)
* フロー図
  * [フロー図1](docs/フロー図/フロー図1.md)
  * [フロー図2](docs/フロー図/フロー図2.md)
* [外部リンク名](https://example.com)
```

**このリポジトリで実装しているカスタムプラグイン（`index.html` 内）:**

| プラグイン | 概要 |
|-----------|------|
| SVG インライン展開 | `<img src="*.svg">` を `<svg>` 要素に置換してリンクを有効化 |
| サイドバー折りたたみ | `_sidebar.md` のネストをフォルダとして開閉できるよう実装 |
| サイドバーリサイズ | ドラッグハンドルでサイドバー幅を変更可能にする |
| grid.js CSV テーブル | コードブロック言語指定 `gridjs` で CSV を検索・ソート付きテーブルとして描画 |
| 外部 URL iframe 表示 | サイドバーの外部リンクをクリックすると右ペイン内 iframe で表示 |

### Draw.io Integration の使い方

VSCode 内で draw.io 図を作成・編集できる拡張機能。

**ファイル形式:**

| 拡張子 | 特徴 |
|--------|------|
| `.drawio` | draw.io 専用形式（XMLベース） |
| `.drawio.png` | PNG画像として保存しつつ、draw.io の編集情報も内包 |
| `.drawio.svg` | SVG形式。Markdown から `![](xxx.drawio.svg)` で直接表示可能 |

**基本操作:**

1. エクスプローラーで右クリック → `新しいファイル` → `ファイル名.drawio.png` などで作成
2. ファイルをクリックすると VSCode 内で draw.io エディタが起動
3. 図を編集後、保存（`Ctrl + S`）するとファイルに反映される

**Markdown への埋め込み:**

`.drawio.png` または `.drawio.svg` は通常の画像として Markdown に貼り込める。

```markdown
![図の説明](images/図.drawio.png)
```

**便利な操作:**

プレビュー画面に表示された draw.io 画像を **右クリック →「画像を開く」** で、該当ファイルの draw.io エディタに直接ジャンプして編集できる。

**リンク付きノードを使う場合:**

draw.io のノードにリンクを設定しても、`.drawio.png` ではリンクが機能しない。**リンクを有効にするには `.drawio.svg` 形式で保存する必要がある。**

**SVGリンクの動作確認について:**

`.drawio.svg` に埋め込んだリンクは、**VSCode のプレビュー上および GitHub のプレビュー上では動作しない**。
リンクの動作確認には **docsify** を使ってローカルでブラウザ経由で表示する形で対応。

さらに、通常の Markdown 画像埋め込み（`![](xxx.drawio.svg)`）のままでは docsify 上でもリンクが機能しなかったため、`index.html` に docsify プラグインを実装することで対応した。

**`index.html` での対応内容（プラグイン）:**

ページ描画のたびに `.svg` 拡張子を持つ `<img>` 要素を検出し、以下の処理を行う。

1. SVG ファイルをテキストとして fetch で読み込み、DOM に変換する
2. SVG 内の `<a>` リンクを走査し、`.md` への相対パスを docsify が認識できるハッシュ形式（`#/xxx.md`）に書き換える
3. 元の `<img>` 要素を実際の `<svg>` 要素に置き換えてインライン展開する

---

**draw.io SVG の英語化（テキスト翻訳）:**

draw.io SVG ファイルには、docsify 上で表示される SVG テキスト要素とは別に、draw.io が編集に使うソースデータが `content` 属性（`<svg content="...">` の形式）に格納されている。このデータは以下の形式でエンコードされており、SVG テキストだけを書き換えても draw.io で開いた際に元の日本語が復元される。

```
content 属性の値 → XML エンコード → base64 デコード → deflate 解凍 → URL エンコードされた mxGraph XML
```

**英語化の手順:**

SVG の表示テキスト（`<text>` タグおよび `<foreignObject>` 内のテキスト）を書き換えるのに加えて、`content` 属性のソースデータも以下の Python スクリプトで更新する。

```python
import base64, zlib

# SVG ファイルから content 属性内の <diagram> タグの値を取り出して encoded に代入
encoded = '（draw.ioのdiagram内のbase64文字列）'

# 1. デコード
xml = zlib.decompress(base64.b64decode(encoded), -zlib.MAX_WBITS).decode('utf-8')

# 2. URL エンコードされた日本語を英語に置換
# （置換テーブルを辞書で定義して .replace() で順次置換）
replacements = {
    '%E5%85%83%E3%81%AE%E7%94%BB%E9%9D%A2': 'Original%20Screen',
    # ... 他のテキストも同様に追加
}
for ja, en in replacements.items():
    xml = xml.replace(ja, en)

# 3. 再エンコード（zlib ヘッダ/チェックサムなし = raw deflate）
compressed = zlib.compress(xml.encode('utf-8'))[2:-4]
new_encoded = base64.b64encode(compressed).decode('utf-8')

# 4. SVG ファイルの content 属性内の旧 encoded を new_encoded に置換して書き戻す
```

**注意点:**

- URL エンコードの対象は `value=` 属性のラベルや `link=` 属性のパスなど。日本語文字は `%XX%XX%XX` 形式で表現されている
- HTML エンティティが混在する場合（例: `&lt;div&gt;` が `%26lt%3Bdiv%26gt%3B` になるなど）は、置換テーブルをそのまま使う
- `link=` 属性のパスも同様に URL エンコードされているため、リンク先のパス変更時は合わせて置換する
- draw.io で直接テキストを編集して再エクスポートする方法でも同じ結果を得られるが、量が多い場合は上記スクリプトの方が効率的

### 外部 Web ページを右ペインに表示する（iframe 埋め込み）

サイドバーに外部 URL のリンクを追加し、クリック時に右ペイン内の `<iframe>` で表示する仕組みを `index.html` のカスタムプラグインで実装している。

**仕組みの概要:**

1. `_sidebar.md` に通常の Markdown リンク記法で外部 URL を記載する

   ```markdown
   * [ページ名](https://example.com)
   ```

2. docsify プラグイン（`hook.doneEach`）でサイドバーの外部リンク（`a[href^="http"]`）にクリックイベントを付与
3. クリック時にコンテンツエリアの内容を `<iframe>` に差し替えて表示

**注意点:**

外部サイトの多くは `X-Frame-Options` または `Content-Security-Policy (CSP)` ヘッダによって iframe 埋め込みを禁止している。その場合、右ペインに表示できない（空白または「接続できません」等のエラーになる）。

| サービス例 | iframe 表示 | 備考 |
|-----------|------------|------|
| Google スプレッドシート（通常 URL） | ✗ 不可 | CSP により拒否される |
| Google スプレッドシート（公開・埋め込み用 URL） | ✓ 可 | `publish?output=html` 形式の URL を使用する |
| GitHub | ✗ 不可 | `X-Frame-Options: DENY` が設定されている |
| 自社・社内システム | サイトによる | iframe 制限がないシステムであれば表示可能 |

**埋め込み可能な URL の調べ方:**

ブラウザの開発者ツール（F12）→「ネットワーク」タブでページ読み込み時のレスポンスヘッダを確認し、`X-Frame-Options` や `Content-Security-Policy` の `frame-ancestors` 設定を確認する。

### CSV 管理（Edit CSV + grid.js）

**Edit CSV 拡張機能**を使って CSV ファイルをスプレッドシート感覚で編集し、**grid.js** でブラウザ上に表テーブルとして表示する構成。

**運用フロー:**

1. CSV ファイルを `data/` フォルダに配置
2. VSCode で CSV ファイルを開くと Edit CSV が自動起動 → セル操作で編集可能
3. Markdown からコードブロック（言語指定 `gridjs`）＋ docsify のファイルインクルード構文で参照

```markdown
[filename](../data/ファイル名.csv ':include :type=code gridjs')
```

4. docsify（`python -m http.server 5000`）でブラウザ表示すると、検索・ソート・ページネーション付きのテーブルとして描画される

**grid.js の読み込みについて:**

`index.html` に `gridjs.umd.js` と CSV パース処理を含むカスタムプラグインを実装している。`docsify-gridjs-plugin` という npm パッケージは存在しないため、index.html 内のプラグインで代替している。

## 3. 設定 (settings.json)

```json
{
  "markdown.copyFiles.destination": {
    "**/*.md": "images/"
  }
}
```

**変更した設定とその理由:**

- `markdown.copyFiles.destination` — Markdown ファイルからクリップボード画像を貼り付けた際の保存先を指定する。
  - キー `"**/*.md"` : 対象ファイルのパターン（すべての .md ファイル）
  - 値 `"images/"` : すべての画像を `images/` フォルダにまとめて保存
  - 例: `画像張り込みテスト.md` に貼り付けた場合 → `images/image.png`
