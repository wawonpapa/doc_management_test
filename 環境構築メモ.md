### Draw.io Integration の使い方

VSCode 内で draw.io 図を作成・編集できる拡張機能。

**ファイル形式:**

| 拡張子 | 特徴 |
|--------|------|
| `.drawio` | draw.io 専用形式（XMLベース） |
| `.drawio.png` | PNG画像として保存しつつ、draw.io の編集情報も内包 |
| `.drawio.svg` | SVG形式。Markdown から `![](xxx.drawio.svg)` で直接表示可能 |

**基本操作:**

1. エクスプローラーで右クリック → `新しいファイル` → `ファイル名.drawio.png` などで作成
2. ファイルをクリックすると VSCode 内で draw.io エディタが起動
3. 図を編集後、保存（`Ctrl + S`）するとファイルに反映される

**Markdown への埋め込み:**

`.drawio.png` または `.drawio.svg` は通常の画像として Markdown に貼り込める。

```markdown
![図の説明](images/図.drawio.png)
```

**便利な操作:**

プレビュー画面に表示された draw.io 画像を **右クリック →「画像を開く」** で、該当ファイルの draw.io エディタに直接ジャンプして編集できる。

### 一つ前のファイルに戻る

「戻る」ではなく **「一つ前のエディタ（ファイル）に切り替える」** 操作を使うのが確実。

| OS | ショートカット |
|----|--------------|
| Windows | `Ctrl + Tab` |
| Mac | `Cmd + Option + ←` |

## 3. 設定 (settings.json)

```json
{
  "markdown.copyFiles.destination": {
    "**/*.md": "images/"
  }
}
```

**変更した設定とその理由:**

- `markdown.copyFiles.destination` — Markdown ファイルからクリップボード画像を貼り付けた際の保存先を指定する。
  - キー `"**/*.md"` : 対象ファイルのパターン（すべての .md ファイル）
  - 値 `"images/"` : すべての画像を `images/` フォルダにまとめて保存
  - 例: `画像張り込みテスト.md` に貼り付けた場合 → `images/image.png`
